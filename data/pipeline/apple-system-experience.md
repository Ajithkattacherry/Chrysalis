## Prep Plan — Jun 15 Round 1 (updated Jun 14)

**Round type:** HM first-round — technical skills discussion + work history, no coding, 30 min Webex
**Webex:** https://appleinc.webex.com/appleinc/j.php?MTID=m08d3c02432d7424ba657a6507dde6cbd

**Team context:** System Experience owns Home Screen, Lock Screen, Notifications, Control Center, Spotlight, Settings, Screen Time, and Core Services (app management, app extension discovery) — the substrate all iOS/macOS apps build on. This team builds APIs other teams depend on, exactly like Ajith's Identity SDK work.

**Core positioning:** "Platform SDK engineer who builds infrastructure other engineers depend on — that's the System Experience responsibility surface. I want to work at the layer where correctness is non-negotiable, at Apple's scale."

### Lead Stories (sharpened for this round)

**Story 1 — Intuit Identity SDK (lead):**
Frame: "My SDK is infrastructure for 3 product teams. When it breaks, three product lines ship the bug. Same quality bar as System Experience." Metrics: 10M+ users, 3 product teams depend on it, full lifecycle ownership (spec, API design, versioning, breaking-change policy).

**Story 2 — Web-to-App SSO / Universal Links + FIDO2:**
Frame: "I've built production features on top of Universal Links and FIDO2 at 2M+ user scale — I know the failure modes of those system primitives, which is why I want to work on the team that owns them." Metrics: 18% sign-in lift, 2M+ users, 0 auth regression.

**Story 3 — TDD / Quality culture:**
Frame: "Quality at Apple scale isn't a phase — it's the engineering culture." 40%→85% coverage, 60% fewer defects. Show you've built quality into a team, not just shipped features.

**Story 4 (back pocket) — SwiftUI migration:**
If asked about modern Swift work: 30+ screens, MVVM-coordinator pattern, tracked WWDC throughout including iOS 27 adaptive layout APIs.

### Why Apple (must land specifically, not generically):
"System Experience specifically. I've spent years at the SDK layer — the layer other engineers depend on. System Experience does that for the entire OS: Home Screen, Notifications, the core services everything else builds on. I know FIDO2 and passkeys deeply, and Apple's push toward Passkeys-everywhere is the direction I've been building toward. I want to work on the team that ships the primitives I've been depending on."

### Questions to ask (top 3):
1. "What's the current charter of this team — has the scope shifted with iOS 27?"
2. "Is System Experience Core Services involved in the Passkeys/Passwords app ecosystem, or is that owned separately?"
3. "What does the first 90 days look like — what would someone in this role own first?"
